Canopy Sight™

Development Plan & Funding Proposal — Build to Full Potential (3–4 Months)

Confidential — for funding and implementation planning. Development and build by internal team; this document covers software scope, timeline, and non-labour costs only.

1. Executive Summary

Canopy Sight is an AI-powered rail and site safety monitoring platform: real-time detection (person, vehicle, animal, equipment, debris), behavioural analytics, incident reconstruction, and operational intelligence. This plan outlines the complete software scope, a 3–4 month development timeline to a sellable product, and all non-labour costs (APIs, CNN training, infrastructure, tools) required to build the system to its full potential. A recommended commercial model is a one-time setup fee (covers development and deployment) plus a monthly subscription to run the platform.

2. Software Scope & Features

2.1 Core Platform (Current & Planned)

  • Dashboard & sites — System metrics, site/device status, GIS-native sites (lat/long, address).
  • Devices & edge agent — Raspberry Pi edge units, camera configs, MeshConnect mesh networking, heartbeat, firmware, stream URLs.
  • AI detection pipeline — YOLO-based object detection (person, vehicle, animal, equipment, debris), ONNX export, zone breach detection, SORT tracking, risk scoring (speed, direction, dwell time, zone, time-of-day).
  • Alerts & notifications — Severity (advisory, warning, critical), status workflow, WebSocket live feed, notification routing (SMS, email, push, webhook).
  • Incidents — Manual incident reports, severity, resolution tracking, contributing conditions, incident reconstruction (timeline of detection events around incident time).
  • Analytics — Occupancy by zone, time-of-day pressure, heatmaps, behavioural patterns (clustering, congestion, loitering), NL query and report generation.
  • Video — Clip metadata, signed URLs (S3/local), playback and retention settings.
  • Security & compliance — Multi-tenant (organization-scoped), Clerk auth, audit logs, RBAC (admin, supervisor, viewer), Helmet, CORS, rate limiting.

2.2 Behavioural & Environmental Use Cases (Full Potential)

  • Early behavioural indicators — Abnormal dwell times (loitering), clustering in waiting areas, pacing/agitation (speed/direction), repeated approach to reception points.
  • Congestion & flow — Overcrowding/pinch points (occupancy thresholds), occupancy-by-zone analytics, time-of-day pressure (peak, out-of-hours, normal).
  • Lone worker & out-of-hours — Time-of-day risk factor (e.g. 22:00–06:00), quiet-zone anomalies, delayed-response metrics.
  • Incident reconstruction & learning — Visual timeline, environmental context, contributing conditions (crowding, zones, layout, time).

2.3 CNN Training Pipeline (Full Potential)

  • Multi-stage YOLO11 pipeline: coarse tune → fine-tune → ultra fine-tune; optional hyperparameter evolution.
  • Classes: person, vehicle, animal, equipment, debris. Dataset: YOLO format (e.g. rail-safety); export to ONNX for edge-agent.
  • Training to “top of range”: multiple dataset variants, evolution runs, and validation to achieve production-grade accuracy.

2.4 Gaps to Close for Full Product Brief

  • Synthetic model layer (model state API, time-series persistence, layer confidence/uncertainty).
  • Deployable sensing ingestion (UAV/sensor/external API ingestion, time-sync, buffered uploads).
  • Operational intelligence: playback page (“what changed”), scenario compare (baseline vs current), reports wired to live API.
