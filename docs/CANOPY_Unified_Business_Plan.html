1. Executive summary

CANOPY. is a software platform that helps organisations understand how risk emerges in complex, changing environments. Existing safety and monitoring systems are largely reactive and compliance-driven: they focus on isolated incidents, static dashboards, and real-time alerts without context. As a result, organisations often know what happened but not how risk formed.

CANOPY. addresses this by focusing on patterns rather than policing. The platform combines deployable sensing, computer vision, and synthetic models to provide continuous situational awareness and to identify patterns that lead to safety incidents before they occur. CANOPY. Sight™ is the web-based product: live and recorded camera views, event and incident timelines, pattern detection (repeat zone entry, dwell, congestion), synthetic site models for spatial context, and playback and analytics across time. It is designed for environments where permanent infrastructure is impractical—rail corridors, warehouses, workshops, and manual-labour sites.

The system is built and delivered by Northern Edge Software, Darius, and CANOPY.—no in-house development team is required. Full deployment and go-live are targeted within 6 months of contract. The platform is operational today, with simulation mode for demonstrations and early pilots underway; access is expanding via request-based onboarding.

2. The opportunity: problem and market

Current safety and monitoring approaches typically focus on isolated incidents, static dashboards, and real-time alerts without context. They struggle to identify long-term risk patterns, adapt to temporary or evolving sites, or explain why incidents keep recurring. CANOPY. is aimed at sectors where this gap is most acute.

Initial target sectors: rail and transport infrastructure; warehouses and logistics; workshops and manual-labour environments. These markets share high safety risk, changing layouts and workflows, temporary operations, and strong regulatory and insurance pressures. CANOPY. is designed to scale across sectors without re-engineering the core system.

3. Solution and differentiation

Rather than monitoring individuals, CANOPY. helps organisations understand how environments create risk. The platform ingests data from deployable cameras and sensors; detects movement, presence, and environmental conditions; models each site as a living synthetic environment; and identifies repeat near-misses, risk hotspots, and behavioural drift over time.

Differentiation rests on three principles:

  1. Patterns over policing — How risk emerges over time, not individual fault.
  2. Synthetic models at the core — Environments are living models, not static dashboards.
  3. Deployable by default — Built for temporary and changing sites, not fixed CCTV.

This positioning reduces surveillance concerns and delivers higher-value insight.

4. Product: CANOPY. Sight™

CANOPY. Sight™ is the web-based platform for viewing and analysing operational environments. It delivers live situational awareness, automated detection and alerting, and time-based playback and analytics, aligned with CANOPY.’s vision of deployable sensing feeding synthetic models with operational intelligence for decision-makers.

Core capabilities

  • Dashboard — Single view of sites, devices, and system health.
  • Sites & devices — Manage locations and edge units (cameras, MeshConnect nodes); monitor status and heartbeats.
  • AI detection — Object detection (person, vehicle, animal) with zone breach detection and risk scoring.
  • Alerts — Severity-based alerts (advisory, warning, critical) with acknowledge/resolve workflow.
  • Playback — Browse events and access time-limited signed links for relevant video clips.
  • Analytics & heatmaps — Date-filtered analytics and spatial heatmaps; pattern detection (repeat zone entry, dwell, congestion).
  • Incidents — Log and track manual incident reports with severity and resolution.
  • Synthetic site models — Spatial context and overlays for playback and review.
  • Multi-tenant & roles — Organisation-scoped data; admin, supervisor, and viewer roles.

The platform supports both real deployments and simulation mode for demonstrations and pilots.

Why choose CANOPY. Sight

  • No black box — Confidence and uncertainty are surfaced; you see what the system knows and how sure it is.
  • Built for operators — Designed for people who need to act, not only for data scientists.
  • API-first & integrable — Dashboards, alerts, and reports can feed existing GIS and ops tools.
  • Edge-ready — Runs on edge hardware (e.g. Raspberry Pi); optional MeshConnect for low-latency wireless backhaul.
  • Offline resilience — Edge agents queue events when connectivity is lost and sync when back online.

5. Commercial model

CANOPY. operates a site-based SaaS subscription model, aligned with how organisations budget for safety and operations. Subscriptions are priced per active site; depth of insight and analysis is reflected in tier choice. Site modelling is offered as a one-time setup service (separate from subscription) to keep SaaS economics clean; setup costs can be amortised over the first 6–12 months where required.

Deployment is designed for rapid rollout, minimal on-site disruption, and temporary or mobile installations. Hardware may be provided by CANOPY., leased, or customer-supplied (existing CCTV where appropriate). The platform is hardware-agnostic.

9. Security and technology

Security is built into the platform and into how we operate it. Production uses Clerk (or equivalent) for authentication; all sensitive API routes require a valid user and organisation. Every query is filtered by organisation ID so tenants cannot access another tenant’s data. Admin, supervisor, and viewer roles control what users can see and do. The API uses security headers (CSP, etc.), CORS is configured for your domains, and rate limiting is applied; secrets are stored in environment variables, not in code. The database is PostgreSQL with Prisma; optional encryption is available for sensitive config (e.g. MeshConnect keys). An audit log records actions, resource type, resource ID, user, IP, and changes for compliance. Raw video is not stored in the database—only metadata (path, duration, size, timestamps); clips live in object storage (e.g. S3). Access to clips is only via time-limited signed URLs after organisation ownership is verified. A more detailed security and data-processing annex can be provided on request (e.g. for GDPR or procurement).

Tech stack: Frontend — Next.js 14, React, TypeScript, Tailwind CSS, ShadCN. Backend — Node.js, Express, tRPC, Prisma, PostgreSQL (optional pgvector). AI/ML — Claude (Anthropic), LangChain, ONNX Runtime, YOLO for detection. Auth — Clerk. Infrastructure — Docker, TurboRepo; CI/CD (e.g. Bitbucket Pipelines); optional Sentry.

Data flow: Cameras and sensors feed an edge agent (e.g. Raspberry Pi) that runs YOLO-based detection, zone checks, and risk scoring, then sends event payloads and optional clip metadata over HTTPS to the central API. Video files are uploaded to object storage; the database stores only paths and metadata. Optional MeshConnect provides mesh networking between edge nodes with encrypted traffic (AES-256). The tRPC API serves the web app and integrations; playback uses short-lived signed URLs after ownership verification.
